# 软件自测与进度跟踪（QC）

## 目的
本目录用于维护软件自测记录和问题进度跟踪文档，供提交 QC 使用。

## 文件说明
- `self-test-record.md`：自测用例与验证结果记录。
- `progress-tracking.md`：问题状态与修复进度跟踪。

## 更新规则
1. 状态变更时，优先更新 `progress-tracking.md`。
2. 每次补丁/新版本验证后，补充或更新 `self-test-record.md`。
3. 责任人、预计完成时间、验证结论需保持一致。
4. 时间统一使用 `YYYY-MM-DD` 格式。

## 当前范围
当前已基于已知 4 个问题完成首版初始化。
